# 图书管理系统 需求分析文档

---

## 1. 引言

### 1.1 编写目的

本文档旨在对"图书管理系统"进行全面的需求分析，明确系统的功能需求、非功能需求、用户角色与用例场景，为系统设计与开发提供依据。

### 1.2 项目背景

随着信息化建设的深入，传统的手工图书管理模式效率低下、数据可追溯性差、逾期管理困难。本项目旨在构建一套面向图书馆管理员与普通借阅用户的在线图书管理系统，实现图书借阅与归还的自动化管理，提升运营效率与用户体验。

### 1.3 适用范围

覆盖图书管理系统的全部功能模块：用户认证与管理、图书信息管理、借阅归还管理、逾期管理、系统参数配置、统计分析、公告管理与操作审计。

---

## 2. 系统概述

### 2.1 系统目标

1. 实现图书借阅与归还流程的自动化处理。
2. 降低人工登记差错率，提高数据可追溯性。
3. 支撑管理员进行统计分析与参数化运营。
4. 保持系统前后端分离架构，便于后续扩展与维护。
5. 提供友好的用户界面与清晰的操作引导。

### 2.2 系统范围

| 领域 | 说明 |
|------|------|
| 用户与认证 | 注册、登录、Token 鉴权、角色隔离 |
| 图书管理 | 图书增删改查、分类管理、上下架 |
| 借阅管理 | 借书、还书、续借、借阅记录查询 |
| 逾期管理 | 逾期扫描、逾期费用计算、逾期记录 |
| 统计报表 | 图书借阅统计、用户借阅统计、概览统计 |
| 系统配置 | 参数化规则配置（借阅天数、最大数量、逾期费用） |
| 操作审计 | 管理员操作日志记录与查询 |
| 公告管理 | 系统公告发布与展示 |

## 4. 功能需求

### 4.1 用户认证模块

**FR-AUTH-01：用户注册**
- 用户可通过用户名、真实姓名、密码、手机号、身份证号进行注册。
- 系统校验用户名唯一性，密码使用 BCrypt 加密存储。
- 新注册用户默认角色为 USER。

**FR-AUTH-02：管理员注册**
- 仅现有管理员可注册新管理员账号，需提供 Token 鉴权。

**FR-AUTH-03：用户登录**
- 支持用户名 + 密码登录。
- 登录成功返回 accessToken + refreshToken + 用户基本信息。
- accessToken 有效期默认 120 分钟，refreshToken 有效期默认 168 小时，均可配置。

**FR-AUTH-04：Token 刷新**
- 使用 refreshToken 获取新的 accessToken。

**FR-AUTH-05：鉴权拦截**
- 受保护接口需携带 `X-Token` 请求头。
- 未登录返回 401，越权返回 403。
- 前端 401 自动清除会话并跳转登录页。

### 4.2 图书管理模块

- **FR-BOOK-01**：管理员可录入书籍信息（书名、作者、出版社、ISBN、分类、库存），ISBN 唯一。
- **FR-BOOK-02**：支持关键词搜索（书名/作者/ISBN/分类）与分页查询，管理端可查全部。
- **FR-BOOK-03**：支持图书更新，库存不可小于当前借出量。
- **FR-BOOK-04**：存在未归还记录时不允许删除图书。
- **FR-BOOK-05**：支持下架/上架操作，下架前确保无未归还记录。
- **FR-BOOK-06**：图书分类支持增删改查，名称唯一。

### 4.3 借阅管理模块

- **FR-BORROW-01**：借阅时校验借阅数量上限（默认 5 本）与库存，原子扣减库存。
- **FR-BORROW-02**：归还时自动计算逾期天数与费用，回补库存，逾期则生成逾期记录。
- **FR-BORROW-03**：续借每本限 1 次，已逾期/已归还不允许续借。
- **FR-BORROW-04**：用户查当前借阅/历史记录；管理员查全部记录，支持按状态和关键词筛选。

### 4.4 逾期管理模块

- **FR-OVERDUE-01**：定时任务自动扫描，将到期的 BORROWED 记录标记为 OVERDUE，默认每日凌晨 3:00 执行。
- **FR-OVERDUE-02**：管理员可手动触发逾期扫描。
- **FR-OVERDUE-03**：管理员查看全部逾期记录，用户查看个人逾期与总罚金。

### 4.5 统计分析模块

- **FR-STAT-01**：管理员仪表盘展示用户数、图书数、借阅数、逾期记录数、在借数。
- **FR-STAT-02**：图书借阅排行（借阅次数/在借次数/归还次数）。
- **FR-STAT-03**：用户借阅排行（借阅次数/在借次数/逾期次数/累计逾期费）。
- **FR-STAT-04**：公共概览（图书总数/在架数/在借数），无需登录。

### 4.6 系统配置模块

- **FR-CONFIG-01**：管理员在线修改借阅天数、最大借阅数量、每日逾期费用，即时生效。

### 4.7 公告管理模块

- **FR-NOTICE-01**：管理员创建/编辑/删除公告，支持发布/取消发布。
- **FR-NOTICE-02**：公开页面展示已发布公告列表与详情，按发布时间倒序。

### 4.8 操作审计模块

- **FR-AUDIT-01**：关键管理操作自动记录日志（操作人、操作名、详情、结果、耗时、IP、UA）。
- **FR-AUDIT-02**：管理员可按操作名称与时间范围筛选查询日志。

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 |
|------|------|
| 常规查询响应 | < 500ms |
| 统计查询响应 | < 2s |
| 并发支持 | 50+ 并发用户 |
| 库存操作 | 原子更新，防超借/负库存 |

### 5.2 安全性需求

| 指标 | 要求 |
|------|------|
| 密码存储 | BCrypt 加密 |
| 认证方式 | JWT 双 Token（accessToken + refreshToken） |
| 角色鉴权 | Spring Security URL 级 + 方法级 |
| CORS | 允许跨域，支持 Cookie 凭证 |
| SQL 注入 | JPA 参数化查询 |

### 5.3 可用性与可维护性

- 前端基于 Element Plus 的现代化 UI，Toast 反馈与二次确认。
- 后端分层架构（Controller-Service-Repository）。
- 前端 Vue 3 组件化 + Pinia 状态管理。
- 系统参数存数据库，环境变量存部署配置，SLF4J 日志分级输出。

## 6. 核心用例描述

### 6.1 用户借阅图书

| 项目 | 描述 |
|------|------|
| 参与者 | 普通用户 |
| 前置条件 | 已登录，有可用借阅额度 |
| 主流程 | 浏览目录 → 选择图书 → 校验库存与上限 → 创建借阅记录 → 原子扣减库存 |
| 异常流程 | 库存不足提示"库存不足，无法借阅"；达到上限提示"已达到最大借阅数量限制" |

### 6.2 用户归还图书

| 项目 | 描述 |
|------|------|
| 参与者 | 普通用户 |
| 前置条件 | 存在未归还记录 |
| 主流程 | 选择归还 → 写入归还时间 → 计算逾期天数与费用 → 回补库存 → 逾期则生成逾期记录 |
| 后置条件 | 记录状态变为 RETURNED，库存增加 1 |

### 6.3 管理员配置系统参数

| 项目 | 描述 |
|------|------|
| 参与者 | 管理员 |
| 前置条件 | 已登录 |
| 主流程 | 进入系统设置 → 修改参数 → 保存 |
| 后置条件 | 参数即时生效，影响后续借阅与费用计算 |
