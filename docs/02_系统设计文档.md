## 第一部分：概要设计

### 1. 系统总体架构

本系统采用**前后端分离**的 B/S 架构，前端为 Vue 3 SPA 应用，后端为 Spring Boot RESTful API 服务，二者通过 HTTP/JSON 协议通信。

```
┌──────────────────────────────────────────────────────┐
│                    客户端（浏览器）                      │
│  ┌─────────────────────────────────────────────────┐ │
│  │          Vue 3 SPA (Vite 构建)                    │ │
│  │  Element Plus UI  │  Pinia 状态管理  │  Vue Router │ │
│  └──────────────────────┬──────────────────────────┘ │
└─────────────────────────┼────────────────────────────┘
                          │ HTTP / JSON (X-Token)
                          ▼
┌──────────────────────────────────────────────────────┐
│                 Spring Boot 后端服务                    │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │
│  │  Controller   │ │   Service     │ │  Repository  │ │
│  │  (REST API)   │→│  (业务逻辑)    │→│  (数据访问)   │ │
│  └──────────────┘ └──────┬───────┘ └──────┬───────┘ │
│                          │                │          │
│  ┌───────────────────────┴────────────────┴────────┐ │
│  │  Spring Security (JWT 认证 + 角色鉴权)           │ │
│  │  GlobalExceptionHandler (统一异常处理)            │ │
│  │  OverdueScheduler (定时逾期扫描)                  │ │
│  └─────────────────────────────────────────────────┘ │
└─────────────────────────┬────────────────────────────┘
                          │ JDBC / JPA
                          ▼
┌──────────────────────────────────────────────────────┐
│              MySQL 8+ / H2 数据库                      │
│  users │ books │ book_categories │ borrow_records    │
│  overdue_records │ system_configs │ admin_operation_logs│
│  notices                                             │
└──────────────────────────────────────────────────────┘
```

## 第二部分：详细设计

### 5. 认证与安全设计

#### 5.1 JWT 认证流程

```
用户登录 ──→ AuthController.login()
              │
              ├─→ UserService.login() 验证用户名密码
              ├─→ AuthService.issueAccessToken() 签发 accessToken (120min)
              ├─→ AuthService.issueRefreshToken() 签发 refreshToken (168h)
              │
              └─→ 返回 { accessToken, refreshToken, user }

后续请求 ──→ TokenAuthenticationFilter (OncePerRequestFilter)
              │
              ├─→ 从 Header 读取 X-Token
              ├─→ AuthService.findValidUser() 解析并验证 Token
              ├─→ 设置 SecurityContext
              └─→ SecurityConfig 进行 URL 级别鉴权

Token 过期 ──→ /api/auth/refresh
              │
              └─→ 用 refreshToken 换取新 accessToken
```

#### 5.2 安全配置

- **密码加密**：BCryptPasswordEncoder，不可逆。
- **Token 签名**：HMAC-SHA256，密钥通过环境变量 `JWT_SECRET` 配置。
- **CORS**：允许所有来源模式，支持凭证传递。
- **CSRF**：禁用（前后端分离通过 Token 保障安全）。
- **Session**：无状态（STATELESS），不创建 HttpSession。
- **URL 权限**：
  - `/api/public/**`、`/api/auth/login`、`/api/auth/register`、`/api/auth/refresh`：公开访问。
  - `/api/admin/**`：需 ADMIN 角色。
  - 其余 `/api/**`：需登录。

### 7. 关键业务流程设计

#### 7.1 借阅流程

```
开始
  │
  ├─→ 1. 校验用户身份 (AuthService.requireUser)
  ├─→ 2. 统计当前借阅数量，与 maxBorrowCount 比较
  │     └─ 超限 → 抛 BusinessException
  ├─→ 3. 查询图书 (BookService.findById)
  │     └─ 不存在 → 抛 BusinessException
  ├─→ 4. 检查 availableStock > 0
  │     └─ 不足 → 抛 BusinessException
  ├─→ 5. 原子扣减库存 (bookRepository.decreaseAvailableStockIfEnough)
  │     └─ UPDATE books SET available_stock = available_stock - 1
  │        WHERE id = :id AND available_stock > 0
  ├─→ 6. 创建 BorrowRecord
  │     ├─ borrowTime = now
  │     ├─ dueTime = now + borrowDays (或自定义期限)
  │     ├─ status = BORROWED
  │     └─ renewCount = 0
  └─→ 7. 保存借阅记录 → 返回
```

#### 7.2 归还流程

```
开始
  │
  ├─→ 1. 校验用户身份与记录所有权
  ├─→ 2. 校验记录状态 ≠ RETURNED
  ├─→ 3. 写入 returnTime = now
  ├─→ 4. 计算逾期天数 = max(0, DAYS(dueTime, returnTime))
  ├─→ 5. 计算逾期费用 = 逾期天数 × overdueDailyFee
  ├─→ 6. 更新记录状态 = RETURNED, overdueFee
  ├─→ 7. 原子回补库存 (bookRepository.increaseAvailableStock)
  ├─→ 8. 若逾期天数 > 0：
  │     └─ 创建 OverdueRecord
  └─→ 9. 保存 → 返回
```

#### 7.3 续借流程

```
开始
  │
  ├─→ 1. 校验用户身份与记录所有权
  ├─→ 2. 校验状态：已归还 → 拒绝
  ├─→ 3. 校验逾期：已逾期(now > dueTime) → 拒绝
  ├─→ 4. 校验续借次数：renewCount >= 1 → 拒绝
  ├─→ 5. 更新 dueTime (后延 borrowDays 或自定义)
  ├─→ 6. renewCount + 1
  └─→ 7. 保存 → 返回
```

#### 7.4 逾期扫描流程（定时任务）

```
定时触发 (cron: 0 0 3 * * ? 每日凌晨3点)
  │
  ├─→ OverdueScheduler.scanOverdue()
  └─→ borrowService.markOverdueRecords()
        │
        └─→ UPDATE borrow_records
            SET status = 'OVERDUE'
            WHERE status = 'BORROWED' AND due_time < NOW()
```

```
  │     └─→ 角色不匹配 → 提示错误 → 跳转 /home
  └─→ next()
```

### 9. 前端状态管理设计

#### 9.1 Auth Store (Pinia)

```javascript
useAuthStore:
  state:
    token        // accessToken (localStorage 持久化)
    refreshToken // 刷新令牌
    user         // 用户信息对象 { id, username, realName, role, ... }
  getters:
    isAuthenticated  // token && user 均存在
  actions:
    initFromStorage()  // 从 localStorage 恢复
    login(credentials) // 登录并持久化
    logout()           // 登出并清除
```

### 10. 异常处理与响应设计

#### 10.1 统一响应格式

```json
{
  "code": "SUCCESS",
  "message": "操作成功",
  "data": { ... },
  "pagination": {
    "page": 0,
    "size": 10,
    "totalElements": 100,
    "totalPages": 10,
    "first": true,
    "last": false
  }
}
```

#### 10.2 错误码定义

| 错误码 | HTTP 状态 | 说明 |
|--------|----------|------|
| SUCCESS | 200 | 成功 |
| AUTH_REQUIRED | 401 | 未登录 |
| ACCESS_DENIED | 403 | 无权限 |
| VALIDATION_ERROR | 400 | 参数校验失败 |
| BUSINESS_ERROR | 400 | 业务逻辑异常 |
| INTERNAL_ERROR | 500 | 服务器内部错误 |

### 11. 前台请求处理设计

#### 11.1 Axios 封装

```
request.js (Axios 实例):
  baseURL: '/api'
  timeout: 30000
  
  请求拦截器:
    └─ 自动附加 X-Token (从 localStorage 读取)
  
  响应拦截器:
    ├─ 兼容两种响应格式 (ApiResponse 与 Map)
    ├─ 401 → 清除 session → 跳转登录页
    ├─ 403 → Toast "权限不足"
    └─ 其他错误 → Toast 错误信息
```

### 12. 系统配置管理设计

#### 12.1 配置存储

| 配置键 | 默认值 | 说明 |
|--------|--------|------|

| max_borrow_count | 5 | 最大借阅数量 |
| overdue_daily_fee | 1.0 | 每日逾期费用（元） |
