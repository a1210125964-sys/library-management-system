## 3. 分层架构详解

### 3.1 Controller 层

**职责**：接收 HTTP 请求，参数校验，调用 Service，封装响应。

**设计原则**：
- 使用 `@RestController` + `@RequestMapping` 定义 API。
- 鉴权通过 `AuthService.requireUser(token)` 或 `AuthService.requireAdmin(token)` 实现。
- 统一使用 `ApiResponse.success()` 返回成功响应。
- 异常由 `GlobalExceptionHandler` 统一处理。

**Controller 清单**：

| Controller | 路径前缀 | 职责 |
|------------|----------|------|
| AuthController | `/api/auth` | 注册、登录、Token 刷新 |
| UserController | `/api/users` | 个人信息查看/修改/改密 |
| UserPortalController | `/api/user` | 用户仪表盘、借阅查询 |
| BookController | `/api/books` | 图书增删改查、上下架 |
| BookCategoryController | `/api/book-categories` | 分类增删改查 |
| BorrowController | `/api/borrow` | 借阅、归还、续借、逾期扫描 |
| AdminController | `/api/admin` | 统计、配置、用户管理、日志、公告、逾期 |
| AdminBorrowRecordController | `/api/admin` | 管理端借阅记录查询 |
| PublicController | `/api/public` | 公共概览、公告、图书目录 |

### 3.2 Service 层

**职责**：实现业务逻辑、编排 Repository 调用、管理事务。

| Service | 核心方法 |
|---------|----------|
| AuthService | `requireUser()`, `requireAdmin()`, `issueAccessToken()`, `issueRefreshToken()`, `refreshAccessToken()` |
| UserService | `register()`, `login()`, `updateProfile()`, `changePassword()`, `resetPassword()`, `updateRole()` |
| BookService | `create()`, `update()`, `delete()`, `shelve()`, `listPage()`, `decreaseAvailableStockOrThrow()`, `increaseAvailableStock()` |
| BorrowService | `borrow()`, `returnBook()`, `renew()`, `markOverdueRecords()`, `listAdminRecords()`, `myRecords()`, `myHistoryPaged()` |
| SystemConfigService | `initDefaults()`, `update()`, `borrowDays()`, `maxBorrowCount()`, `overdueDailyFee()` |
| StatisticsService | `overview()`, `publicOverview()`, `bookBorrowStats()`, `userBorrowStats()` |
| JwtService | `createAccessToken()`, `createRefreshToken()`, `parseAccessTokenUserId()`, `parseRefreshTokenUserId()` |
| AdminLogService | `log()`, `searchLogs()` |
| NoticeService | `create()`, `update()`, `delete()`, `publish()`, `listPublished()` |
| BookCategoryService | `create()`, `list()`, `update()`, `delete()`, `ensureExists()` |
| OverdueScheduler | `scanOverdue()` —— `@Scheduled` 定时任务 |

### 3.3 Repository 层

**职责**：定义 JPA 数据访问接口，提供查询方法。

所有 Repository 继承 `JpaRepository<T, Long>`，通过方法命名约定或 `@Query` 注解定义查询。

**核心 Repository 方法**：

| Repository | 关键方法 |

| UserRepository | `findByUsername()`, `existsByUsername()`, `findByRoleOrderByCreatedAtDesc()` |

## 4. 安全设计

### 4.1 认证流程

```
请求 → TokenAuthenticationFilter (从 X-Token 解析用户)
     → SecurityContextHolder (设置认证信息)
     → SecurityFilterChain (URL 级鉴权)
     → Controller (AuthService 手动鉴权)
```

### 4.2 TokenAuthenticationFilter

- 继承 `OncePerRequestFilter`，每个请求执行一次。
- 从 Header 读取 `X-Token`。
- 调用 `AuthService.findValidUser(token)` 解析 Token 并查询用户。
- 若有效，设置 `SecurityContext` 为用户认证信息。
- 角色以 `ROLE_USER` / `ROLE_ADMIN` 形式注入。

### 4.3 SecurityConfig

**CORS 配置**：
- 允许所有来源（`setAllowedOriginPatterns(List.of("*"))`）。
- 允许 GET / POST / PUT / DELETE / OPTIONS。
- 允许凭证传递。

**URL 权限**：
```
/api/public/**                 → permitAll (公开)
/api/auth/login|register|refresh → permitAll (公开)
/api/book-categories           → permitAll (公开)
/api/admin/**                  → hasRole('ADMIN')
/api/**                        → authenticated (需登录)
其他                            → denyAll
```

**异常处理**：
- `authenticationEntryPoint`：未认证返回 401 + `AUTH_REQUIRED` 错误码。
- `accessDeniedHandler`：无权限返回 403 + `ACCESS_DENIED` 错误码。

### 4.4 JwtService

- **签名算法**：HMAC-SHA256。
- **密钥**：通过环境变量 `JWT_SECRET` 配置，不足 32 位自动补零。
- **AccessToken**：包含 sub(userId)、username、role、type="access"、exp。
- **RefreshToken**：包含 sub(userId)、type="refresh"、exp。
- **Token 类型校验**：解析时校验 `type` 字段，防止 Token 混用。

## 5. 核心业务逻辑

### 5.1 借阅并发安全

```java
// BookRepository
@Modifying
@Query("UPDATE Book b SET b.availableStock = b.availableStock - 1 WHERE b.id = :id AND b.availableStock > 0")
int decreaseAvailableStockIfEnough(@Param("id") Long id);
```

使用 `AND b.availableStock > 0` 条件实现原子更新，受影响行数为 0 时抛异常，防止超借与负库存。

### 5.2 逾期费用计算

```java
long overdueDays = Math.max(0, ChronoUnit.DAYS.between(dueTime, returnTime));
BigDecimal fee = overdueDailyFee × overdueDays;
```

逾期天数取应还日与归还日之间的整天数，最小为 0。费用 = 天数 × 日单价。

### 5.3 定时逾期扫描

```java
@Scheduled(cron = "${app.borrow.overdue-scan-cron:0 0 3 * * ?}")
public void scanOverdue() {
    int updated = borrowService.markOverdueRecords();
}

// 批量更新
@Modifying
@Query("UPDATE BorrowRecord br SET br.status = :toStatus WHERE br.status = :fromStatus AND br.dueTime < :now")
int markOverdue(...);
```

默认每日凌晨 3:00 执行，批量标记到期的 BORROWED 记录为 OVERDUE。

### 5.4 事务管理

- 借阅、归还、续借操作使用 `@Transactional` 保证原子性。
- 库存扣减/回补在同一个事务内完成。
- 逾期记录生成在归还事务内同步完成。

## 6. 异常处理设计

### 6.1 异常体系

```
Exception
  ├── BusinessException          → 业务异常，返回 400 + BUSINESS_ERROR
  │   (如：库存不足、用户名已存在)
  │
  └── (其他异常)                 → GlobalExceptionHandler 统一处理
```

### 6.2 GlobalExceptionHandler

| 异常类型 | HTTP 状态 | 错误码 |
|----------|----------|--------|
| BusinessException | 400 | BUSINESS_ERROR |
| MethodArgumentNotValidException | 400 | VALIDATION_ERROR |
| MissingRequestHeaderException (X-Token) | 401 | AUTH_REQUIRED |
| MethodArgumentTypeMismatchException | 400 | VALIDATION_ERROR |
| Exception (兜底) | 500 | INTERNAL_ERROR |

### 6.3 统一响应格式

```java
public class ApiResponse<T> {
    private String code;          // SUCCESS
    private String message;       // "操作成功"
    private T data;               // 业务数据
    private Map<String, Object> pagination;  // 分页信息
}
```

## 7. 数据初始化设计

### 7.1 DataInitializer

实现 `CommandLineRunner`，在应用启动时执行：

1. 调用 `SystemConfigService.initDefaults()` 初始化配置项。
2. 创建默认管理员账户（admin / admin123），若不存在。
3. 插入 8 本示例计算机类图书，若图书表为空。
4. 确保所有图书分类已存在。
