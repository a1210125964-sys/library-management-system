## 3. 数据库配置

### 3.1 开发模式（H2 内嵌数据库）

默认配置即可直接使用，无需额外安装数据库。H2 数据库文件存储在 `./data/` 目录。

**配置文件** (`backend/main/resources/application.properties`)：

```properties
# H2 内嵌数据库
spring.datasource.url=jdbc:h2:file:./data/library_management;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE;MODE=MySQL
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console
spring.jpa.hibernate.ddl-auto=update
spring.sql.init.mode=never
```

**访问 H2 控制台**：`http://localhost:8080/h2-console`

### 3.2 生产模式（MySQL）

#### Step 1：创建数据库

```sql
CREATE DATABASE library_management DEFAULT CHARSET utf8mb4;
```

#### Step 2：配置环境变量

**Windows PowerShell**：

```powershell
$env:DB_USERNAME="root"
$env:DB_PASSWORD="your_password"
```

**Linux / macOS**：

```bash
export DB_USERNAME=root
export DB_PASSWORD=your_password
```

#### Step 3：切换配置文件

在 `application.properties` 中注释掉 H2 配置，取消注释 MySQL 配置：

```properties
# MySQL 配置
spring.datasource.url=jdbc:mysql://localhost:3306/library_management?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
spring.datasource.username=${DB_USERNAME:root}
spring.datasource.password=${DB_PASSWORD:123456}
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.sql.init.mode=${SQL_INIT_MODE:always}
spring.jpa.hibernate.ddl-auto=${JPA_DDL_AUTO:validate}
```

## 4. 后端启动

### 4.1 编译

```bash
mvn clean compile
```

### 4.2 运行测试

```bash
mvn test
```

### 4.3 启动服务

```bash
# 方式 A：Maven 插件启动
mvn spring-boot:run

# 方式 B：打包 JAR 后运行
mvn clean package -DskipTests
java -jar target/library-management-system-1.0.0.jar
```

### 4.4 启动验证

服务启动后，访问 `http://localhost:8080` 应返回 API 响应。

## 5. 前端启动

### 5.1 安装依赖

```bash
cd frontend
npm install
```

### 5.2 开发模式启动

```bash
npm run dev
```

前端开发服务器运行在 `http://localhost:5173`，API 请求通过 Vite 代理转发到后端 `localhost:8080`。

### 5.3 生产构建

```bash
npm run build
```

构建产物在 `frontend/dist/` 目录，可部署到 Nginx 或放入后端 `backend/main/resources/static/` 作为静态资源。

## 6. 环境变量配置

### 6.1 环境变量清单

以下环境变量应在生产环境中配置：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `DB_USERNAME` | root | MySQL 数据库用户名 |
| `DB_PASSWORD` | - | MySQL 数据库密码 |
| `JWT_SECRET` | (默认密钥) | **必须修改！** 至少 32 位随机字符串 |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | 120 | AccessToken 有效期（分钟） |
| `REFRESH_TOKEN_EXPIRE_HOURS` | 168 | RefreshToken 有效期（小时） |
| `OVERDUE_SCAN_CRON` | 0 0 3 * * ? | 逾期扫描定时任务 cron 表达式 |
| `JPA_DDL_AUTO` | validate | JPA DDL 策略（生产建议 validate） |
| `SQL_INIT_MODE` | always | SQL 初始化模式（首次 always，之后 never） |
| `LOG_LEVEL_ROOT` | INFO | 根日志级别 |
| `LOG_LEVEL_APP` | INFO | 应用日志级别 |
| `SERVER_PORT` | 8080 | 服务端口 |

### 6.2 安全提醒

> ⚠️ **生产环境必须修改 `JWT_SECRET`**，使用至少 32 位高强度随机字符串。
>
> ⚠️ 默认管理员密码 `admin123` 应在首次登录后立即修改。

## 9. 常见问题

### Q1：H2 数据库文件在哪里？

A：默认在项目根目录 `./data/library_management.mv.db`。如需重置，删除 `data` 文件夹后重启。

### Q2：如何切换 H2 / MySQL？

A：修改 `application.properties` 中的数据库配置，注释/取消注释对应区块即可。

### Q3：前端代理不生效？

A：确保后端在 8080 端口运行。检查 `vite.config.js` 中的 proxy 配置。

### Q4：401 错误怎么办？

A：Token 过期或未登录。刷新页面或重新登录。

### Q5：忘记管理员密码？

A：开发环境可删除 `data/` 文件夹后重启（H2），系统会重新创建默认管理员。生产环境需通过数据库直接修改 BCrypt 密文。
