## 4. 路由设计

### 4.1 路由架构

采用三层布局嵌套路由：

```
App.vue
  └── <router-view>
        ├── PublicLayout   → path: /       → 公共页面布局
        ├── UserLayout     → path: /user   → 用户页面布局 (需 USER 角色)
        └── AdminLayout    → path: /admin  → 管理页面布局 (需 ADMIN 角色)
```

### 4.2 路由表

**公共路由 (PublicLayout)**

| 路径 | 组件 | 说明 |

| `/` | redirect → /home | 默认跳转 |

### 4.3 导航守卫

```javascript
router.beforeEach((to, from, next) => {
  const auth = useAuthStore()
  auth.initFromStorage()           // 从 localStorage 恢复登录态
  
  if (to.meta.requiresAuth) {
    if (!auth.isAuthenticated) {   // 未登录
      ElMessage.warning('请先登录')
      return next('/login')
    }
    if (roles不匹配) {             // 角色越权
      ElMessage.error('权限不足')
      return next('/home')
    }
  }
  next()
})
```

## 5. 状态管理设计

### 5.1 Auth Store (Pinia)

使用 Pinia 的 Composition API 风格定义：

```javascript
useAuthStore:
  state:
    token         - accessToken 字符串
    refreshToken  - 刷新令牌
    user          - { id, username, realName, role, phone, idCard }
  
  getters:
    isAuthenticated - token 与 user 均非空时为 true
  
  actions:
    initFromStorage()    - 从 localStorage 恢复状态
    login(credentials)   - 登录 → 保存 token 与 user 到 localStorage
    logout()             - 登出 → 清除 localStorage
```

### 5.2 持久化策略

- `token`、`refreshToken`、`user` 均以 JSON 字符串形式存储于 localStorage。
- 页面刷新时通过 `initFromStorage()` 恢复，保证会话不丢失。
- 登出时调用 `logout()` 清除全部 localStorage 数据。

## 6. HTTP 请求封装

### 6.1 Axios 实例 (request.js)

```
baseURL: '/api'
timeout: 30s
headers: { 'Content-Type': 'application/json' }
```

### 6.2 请求拦截器

```
request → 从 localStorage 读取 token
         → 自动设置 X-Token 请求头
         → 发送请求
```

### 6.3 响应拦截器

```
response → 兼容两种后端响应格式:
           格式A: { code, message, data, pagination }    (ApiResponse)
           格式B: { message, data, pagination }          (Map)
         → 统一返回 { code, message, data, pagination }

error    → 401: 清除 localStorage → 提示 → 跳转 /login
         → 403: Toast "权限不足"
         → 其他: Toast 后端错误信息
         → 网络异常: Toast "网络异常，请检查连接"
```

### 6.4 API 模块函数示例

```javascript
// auth.js
export function login(data)     → POST /api/auth/login
export function register(data)  → POST /api/auth/register
export function refreshToken(t) → POST /api/auth/refresh

// borrow.js
export function borrowBook(id, dueTime) → POST /api/borrow/{id}
export function returnBook(id)          → POST /api/borrow/return/{id}
export function renewBook(id, dueTime)  → POST /api/borrow/renew/{id}
export function getMyBorrowings()       → GET  /api/borrow/my
export function getMyOverdues()         → GET  /api/borrow/my-overdue

// admin.js
export function getStats()             → GET  /api/admin/stats
export function listUsers({...})       → GET  /api/admin/users
export function listLogs({...})        → GET  /api/admin/logs
export function getConfigs()           → GET  /api/admin/configs
export function updateConfigs(data)    → PUT  /api/admin/configs
export function listRecords({...})     → GET  /api/admin/records
export function scanOverdue()          → POST /api/borrow/scan-overdue
// ... 共 16 个函数
```

## 7. 布局组件设计

### 7.1 PublicLayout

公共页面的外层布局，包含顶部导航栏与底部信息栏，导航菜单包括：首页、图书目录、公告、关于、登录/注册。

### 7.2 UserLayout

用户页面的外层布局，包含侧边栏导航（仪表盘、当前借阅、借阅历史、逾期罚金、个人信息）与顶部用户信息栏。

### 7.3 AdminLayout

管理员页面的外层布局，包含侧边栏导航（仪表盘、图书管理、用户管理、借阅记录、逾期管理、公告管理、系统设置、操作日志）与顶部管理员信息栏。

## 8. 页面组件设计要点

### 8.1 用户端页面

| 页面 | 核心功能 |
|------|----------|
| Dashboard | 展示在借数量、逾期数量、公告数 |

## 9. UI 交互规范

| 规范项 | 说明 |
|--------|------|
| 表单校验 | 用户名长度、手机号格式、密码长度前端校验 |
| 操作反馈 | 成功/失败统一使用 `ElMessage` Toast 提示 |
| 删除确认 | 关键删除操作使用 `ElMessageBox.confirm` 二次确认 |
| 空状态 | 列表无数据时显示引导式空状态提示 |
| 加载状态 | 使用 `v-loading` 指令展示加载中 |
| 分页 | 统一使用 `el-pagination` 组件 |
| 低库存预警 | 可借库存低于阈值时红色高亮 |
