## 3. 技术架构总结

### 3.1 后端架构

```
Controller (9个) → Service (11个) → Repository (8个) → Model (10个实体 + 2个枚举)
                    ↑
              Config (4个) + Exception (4个)
```

**核心设计模式**：
- 分层架构：Controller - Service - Repository - Model
- DTO 模式：请求体与响应体分离
- 统一响应：`ApiResponse<T>` + `ErrorCode` 枚举
- 全局异常处理：`@RestControllerAdvice`
- 事务管理：`@Transactional` 保证复合操作原子性
- 原子更新：库存扣减使用条件 UPDATE 防并发问题

### 3.2 前端架构

```
Views (20个页面) → API (8个模块) → Axios 实例 (统一拦截)
       ↕                        ↕
Layout (3个)          Stores (auth.js)
       ↕
Router (导航守卫 + 角色鉴权)
```

**核心设计模式**：
- SPA 单页应用 + 组件化开发
- 路由级角色控制（导航守卫 `beforeEach`）
- 三层布局嵌套（Public / User / Admin）
- 请求/响应统一拦截（401 自动跳转登录）
- Pinia 状态集中管理（localStorage 持久化）

### 3.3 数据库架构

共 8 张业务表：users、books、book_categories、borrow_records、overdue_records、system_configs、admin_operation_logs、notices。

核心关系：
- users ←── borrow_records ──→ books ←─ book_categories
- borrow_records ──→ overdue_records
- users ──→ admin_operation_logs

## 5. 关键技术亮点

### 5.1 JWT 双 Token 机制

- AccessToken 短期有效（默认 120 分钟），用于 API 鉴权。
- RefreshToken 长期有效（默认 168 小时），用于刷新 AccessToken。
- Token 类型严格校验，防止 Access/Refresh 混用。

### 5.2 库存并发安全

```sql
UPDATE books SET available_stock = available_stock - 1
WHERE id = ? AND available_stock > 0
```

通过条件 UPDATE 实现乐观锁，避免超借与负库存。

### 5.3 参数化系统配置

借阅天数、最大借阅数量、逾期日费通过数据库 `system_configs` 表管理，支持管理员在线修改即时生效，避免硬编码。

### 5.4 角色化前端路由

通过 Vue Router 导航守卫 + 三层 Layout 实现前端路由级别的角色隔离，保证 USER 无法访问管理页面。

### 5.5 统一异常处理

- `BusinessException` 统一业务异常。
- `GlobalExceptionHandler` 全局异常处理，规范化错误响应。
- 前端 Axios 拦截器统一处理 401/403。

### 5.6 管理操作审计

关键管理操作（修改配置、变更角色、图书管理等）自动记录：操作人、操作名、结果、耗时、IP、UA，支持按条件检索。

## 6. 不足与改进方向

| 序号 | 不足 | 改进建议 |
|:---:|------|----------|
| 1 | 库存并发在极高并发下可能存在竞争 | 引入数据库悲观锁或分布式锁（Redis） |
| 2 | 无自动化 CI/CD | 接入 GitHub Actions 或 Jenkins |
| 3 | 测试覆盖率不够全面 | 补充更多边界条件测试 |
| 4 | 无消息通知机制 | 扩展逾期提醒（短信/邮件/站内信） |
| 5 | 无报表导出功能 | 增加 Excel/CSV 导出 |
| 6 | Token 无黑名单机制 | 引入 Redis 管理 Token 黑名单 |
| 7 | 无接口限流 | 引入 Rate Limiter 防止 API 滥用 |
| 8 | 前端无国际化 | 引入 vue-i18n 支持多语言 |

## 8. 结论

本图书管理系统已实现完整的业务闭环：

- **用户端**：注册、登录、个人信息维护、图书浏览、借阅、归还、续借、借阅历史查看、逾期罚金查询。
- **管理端**：图书与分类管理、用户管理、借阅记录管理、逾期管理、统计报表、系统参数配置、公告管理、操作审计。
- **系统端**：JWT 双 Token 认证、角色级权限隔离、并发安全的库存管理、定时逾期扫描、统一异常处理。

系统已具备投入教学演示与中小规模业务验证的能力，同时为后续的性能优化与功能扩展奠定了良好的架构基础。
