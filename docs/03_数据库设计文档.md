# 图书管理系统 数据库设计文档

---

## 1. 数据库概述

### 1.1 数据库环境

| 项目 | 开发环境 | 生产环境 |
|------|----------|----------|
| 数据库类型 | H2 (File Embedded) | MySQL 8+ |
| 连接模式 | 本地文件 | TCP 远程连接 |
| ORM 框架 | Spring Data JPA (Hibernate) | 同左 |
| DDL 策略 | update (自动建表) | validate (手动管理) |
| 字符集 | - | utf8mb4 |

### 1.2 命名规范

- 表名：小写 + 下划线分隔，复数形式（如 `users`、`borrow_records`）。
- 字段名：小写 + 下划线分隔（如 `real_name`、`available_stock`）。
- 主键：统一使用 `id BIGINT AUTO_INCREMENT`。
- 外键：`关联表名单数_id`（如 `user_id`、`book_id`）。
- 索引：`idx_表名_字段名`（如 `idx_borrow_user_status`）。

## 3. 核心表结构设计

### 3.1 users —— 用户表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 用户唯一标识 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| real_name | VARCHAR(50) | NOT NULL | 真实姓名 |
| phone | VARCHAR(20) | - | 手机号 |
| id_card | VARCHAR(30) | - | 身份证号 |

| active | BOOLEAN | NOT NULL, DEFAULT TRUE | 是否在架 |

| updated_at | DATETIME | NOT NULL | 更新时间 |

**索引**：`idx_books_category(category)`

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|

| 字段名 | 类型 | 约束 | 说明 |

| id | BIGINT | PK, AUTO_INCREMENT | 记录 ID |

## 5. 关键 SQL 设计

### 5.1 库存原子扣减

```sql
UPDATE books 
SET available_stock = available_stock - 1 
WHERE id = ? AND available_stock > 0
```

通过 `available_stock > 0` 条件实现乐观锁式的并发控制，返回受影响行数为 0 时说明库存不足。

### 5.2 库存原子回补

```sql
UPDATE books 
SET available_stock = available_stock + 1 
WHERE id = ?
```

归还时单向增加，无并发竞争风险。

### 5.3 批量逾期标记

```sql
UPDATE borrow_records 
SET status = 'OVERDUE' 
WHERE status = 'BORROWED' AND due_time < NOW()
```

定时任务每日凌晨执行，将到期未还的借阅记录标记为逾期。

### 5.4 图书借阅统计

```sql
SELECT 
    b.id, b.title, b.author, b.category,
    COUNT(br.id) AS borrow_count,
    SUM(CASE WHEN br.status IN ('BORROWED','OVERDUE') THEN 1 ELSE 0 END) AS active_borrow_count,
    SUM(CASE WHEN br.status = 'RETURNED' THEN 1 ELSE 0 END) AS returned_count
FROM borrow_records br
JOIN books b ON b.id = br.book_id
GROUP BY b.id, b.title, b.author, b.category
ORDER BY borrow_count DESC
LIMIT ?
```

### 5.5 用户借阅统计

```sql
SELECT 
    u.id, u.username, u.real_name,
    COALESCE(brs.borrow_count, 0) AS borrow_count,
    COALESCE(brs.active_borrow_count, 0) AS active_borrow_count,
    COALESCE(ors.overdue_count, 0) AS overdue_count,
    COALESCE(ors.overdue_fee_total, 0) AS overdue_fee_total
FROM users u
LEFT JOIN (
    SELECT user_id, COUNT(*) AS borrow_count,
        SUM(CASE WHEN status IN ('BORROWED','OVERDUE') THEN 1 ELSE 0 END) AS active_borrow_count
    FROM borrow_records GROUP BY user_id
) brs ON brs.user_id = u.id
LEFT JOIN (
    SELECT user_id, COUNT(*) AS overdue_count,
        SUM(overdue_fee) AS overdue_fee_total
    FROM overdue_records GROUP BY user_id
) ors ON ors.user_id = u.id
WHERE u.role = 'USER'
ORDER BY borrow_count DESC, overdue_count DESC
LIMIT ?
```

### 5.6 管理员借阅记录搜索

```sql
SELECT br FROM BorrowRecord br
JOIN br.user u JOIN br.book b
WHERE (:status IS NULL OR br.status = :status)
  AND (:keyword IS NULL 
       OR LOWER(u.username) LIKE CONCAT('%', :keyword, '%')
       OR LOWER(u.realName)  LIKE CONCAT('%', :keyword, '%')
       OR LOWER(b.title)    LIKE CONCAT('%', :keyword, '%'))
ORDER BY br.borrowTime DESC
```

## 6. 数据初始化策略

### 6.1 启动时初始化

通过 `DataInitializer`（实现 `CommandLineRunner`）在应用启动时执行：

1. **系统配置初始化**：写入 borrow_days / max_borrow_count / overdue_daily_fee 默认值（若不存在）。
2. **默认管理员创建**：用户名 admin，密码 admin123（若不存在）。
3. **示例图书数据**：8 本计算机类书籍（Java 核心技术、深入理解计算机系统、数据库系统概念等）。
4. **分类初始化**：编程、计算机基础、数据库、算法、软件工程、前端。

### 6.2 配置文件方式

- 默认值通过 `application.properties` 中的 `app.borrow.*` 配置项指定。
- 通过 `spring.sql.init.mode=always` 可启用 schema.sql 自动建表。

## 7. 数据库迁移

### 7.1 开发模式 (H2)

- `spring.jpa.hibernate.ddl-auto=update`：自动根据 Entity 更新表结构。
- `spring.sql.init.mode=never`：不执行 schema.sql。

### 7.2 生产模式 (MySQL)

- `spring.jpa.hibernate.ddl-auto=validate`：仅校验表结构与 Entity 是否一致。
- `spring.sql.init.mode=always`：首次启动执行 schema.sql 建表。
- schema.sql 中使用 `CREATE TABLE IF NOT EXISTS` 确保幂等。
- 新增字段通过动态 ALTER TABLE 方式兼容旧表。
