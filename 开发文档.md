## 1. 项目概述

图书管理系统面向图书馆管理员与普通借阅用户，围绕“人、书、借阅记录、规则参数”四个核心对象，提供注册登录、图书录入、借阅归还、逾期处理、统计分析与系统参数配置能力。

### 1.1 建设目标

1. 提升图书借阅与归还流程自动化水平。
2. 降低人工登记差错并提高数据可追溯性。
3. 支撑管理员进行统计分析与配置化运营。
4. 保持系统结构清晰，便于后续扩展。

### 1.2 角色划分

- **普通用户（USER）**：登录、维护个人信息、借阅/续借/归还、查看个人逾期记录。
- **管理员（ADMIN）**：图书与分类管理、用户管理、系统参数配置、借阅统计查看、逾期扫描。

## 3. 系统设计

## 3.1 总体架构

采用典型分层架构：

- **表现层（Controller）**：处理 HTTP 请求、鉴权、参数校验、响应封装。
- **业务层（Service）**：实现借阅规则、库存更新、费用计算、统计聚合。
- **数据访问层（Repository）**：基于 Spring Data JPA 持久化与聚合查询。
- **数据层（MySQL）**：保存业务主数据与日志数据。

### 3.2 核心业务对象

- 用户（users）
- 图书（books）
- 图书分类（book_categories）
- 借阅记录（borrow_records）
- 逾期记录（overdue_records）
- 系统参数（system_configs）
- 管理员操作日志（admin_operation_logs）

### 3.3 关键业务流程设计

#### A. 借阅流程

1. 校验用户身份与借阅上限。
2. 校验图书库存可借。
3. 创建借阅记录，写入借阅时间与应还时间。
4. 扣减可借库存。

#### B. 归还流程

1. 校验归还权限与记录状态。
2. 写入归还时间并计算逾期天数。
3. 根据规则计算逾期费用。
4. 回补库存，并更新借阅记录状态。
5. 若逾期则新增逾期记录。

#### C. 统计流程

1. 基于借阅记录与逾期记录进行聚合。
2. 产出图书维度、用户维度统计视图。
3. 管理员端加载统计表用于运营分析。

## 4. 开发关键技术

### 4.1 后端技术

- **Spring Boot**：快速搭建 REST 服务。
- **Spring Data JPA**：实体映射与数据访问。
- **事务管理（@Transactional）**：保证借阅、归还等复合操作一致性。
- **Bean Validation（jakarta.validation）**：接口入参校验。
- **Spring Security PasswordEncoder**：密码加密存储。

### 4.2 前端技术

- 原生 **HTML + CSS + JavaScript**。
- 基于 `fetch` 封装统一请求函数，携带 `X-Token`。
- 管理端与用户端按角色动态展示页面模块。
- 任务栏按钮与页面 ID 一一映射，点击按钮直接切换至对应模块页面。
- 新增统计页面区块与系统参数配置区块。

### 4.3 数据与规则设计

- 通过 `system_configs` 实现规则参数化，避免硬编码。
- 逾期费用计算遵循：

$$
逾期费用 = 逾期天数 \times 每日逾期费用
$$

- 统计查询通过聚合 SQL 实现高效排行与汇总。

### 4.4 工程质量实践

- 统一异常处理与业务异常返回。
- 关键管理员操作写入操作日志，支持审计。
- 每次改动后进行编译校验，确保可构建。

## 7. 风险与改进建议

1. **并发库存一致性**：高并发借阅场景可进一步引入悲观/乐观锁策略。
2. **自动化测试**：建议补充单元测试与集成测试，提高回归效率。
3. **告警能力**：可扩展短信/邮件提醒与消息中心。
4. **报表导出**：可增加统计报表导出（Excel/CSV）。
5. **安全增强**：可补充 refreshToken 轮换、token 黑名单与接口限流。

本阶段开发已完成核心业务闭环：

- 用户可维护个人信息；
- 管理员可进行图书录入与系统参数管理；

本次进一步完成了“按角色分离任务栏 + 模块入口一一对应”的交互规范落地：

- 用户端界面聚焦借阅查询与还书操作；
- 管理员端集中呈现学生、图书、管理员与密码管理模块；
- 权限展示与页面入口自动匹配登录角色，降低误操作风险。

系统已具备投入教学演示与中小规模业务验证的能力，并为下一阶段性能优化与自动化测试奠定基础。
